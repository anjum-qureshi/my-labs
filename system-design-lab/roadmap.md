# System Design Roadmap

## Goal
Become interview-ready for backend/system design rounds in 4–8 weeks.

---

# Phase 1 — Foundations
Goal: Understand core building blocks.

Topics:
- Load Balancing
- Caching
- SQL vs NoSQL
- Basic scalability

Problems:
- URL Shortener
- Logging System

---

# Phase 2 — Data Flow Systems
Goal: Learn async systems and storage flow.

Topics:
- Messaging Queues
- Rate Limiting
- Hot vs Cold Storage

Problems:
- Chat System
- Notification System
- File Storage

---

# Phase 3 — Scaling Systems
Goal: Handle scale and bottlenecks.

Topics:
- Sharding
- Replication
- Feed generation
- Caching strategies

Problems:
- News Feed
- Ride Sharing
- Logging System (redo)

---

# Phase 4 — Interview Readiness
Goal: Speak clearly and structure answers.

Tasks:
- Timed mock interviews
- Redraw systems from memory
- Tradeoff discussions


📅 4-Week System Design Plan (Interview-focused)
🟢 Week 1: Foundations (components)

🎯 Goal: know building blocks

Learn:
load balancer
caching (Redis concept)
SQL vs NoSQL basics
basic system flow
Problems:
URL Shortener (basic)
Logging system (basic)
Output:

can draw simple architecture without confusion

🟡 Week 2: Core system thinking

🎯 Goal: connect components

Learn:
messaging queues (Kafka idea)
rate limiting
storage basics (hot vs cold)
Problems:
Chat system
Notification system
File storage system
Output:

can explain full data flow clearly

🟠 Week 3: Scaling & tradeoffs

🎯 Goal: “what breaks at scale”

Learn:
caching strategies
sharding basics
replication basics
Problems:
News feed system
Ride sharing system (basic version)
redo Logging system (better version)
Output:

can explain tradeoffs confidently

🔴 Week 4: Interview simulation

🎯 Goal: performance under pressure

Daily (1 system per day):
URL shortener (redo)
Chat system (redo)
News feed (redo)
Logging system (redo)
File storage (redo)
Rules:
45–60 min timer
speak out loud
no notes
Output:

structured interview answers

🔁 Weekly revision system (VERY important)
Saturday
pick 2 systems
redraw from memory
fix missing pieces (storage, flow, tradeoffs)
Sunday
mock interview simulation
no notes
explain out loud like real interview
🧠 Daily 1-hour structure
10 min → revise yesterday
20 min → learn concept
20 min → apply to system
10 min → write mistakes + tradeoffs
⚠️ Non-negotiable rules
Don’t over-read theory
Always apply immediately
Always include storage + flow (no skipping)
Always mention tradeoffs (this is where seniors stand out)
